# Hooks Reference — @lanonasis/claude-memory

## Claude Code Hook Events

Claude Code fires lifecycle events and passes them to registered hook scripts via stdin (JSON). The hooks registered by this package are:

| Event | Script | Purpose |
|---|---|---|
| `Stop` | `~/.lanonasis/hooks/stop.sh` | End-of-session memory capture |
| `SubagentStop` | `~/.lanonasis/hooks/subagent_stop.sh` | Subagent completion capture |
| `PreCompact` | `~/.lanonasis/hooks/precompact.sh` | Pre-compaction strict capture |
| `PreToolUse` | `~/.lanonasis/hooks/pretooluse.sh` | First-call recall injection |

---

## StopInput Schema

The `Stop` and `SubagentStop` hooks receive this JSON on stdin:

```typescript
type StopInput = {
  session_id: string;          // Unique session identifier
  transcript: TranscriptMessage[];  // Full conversation turns
  // ... additional Claude Code fields
};

type TranscriptMessage = {
  role: "user" | "assistant";
  content: string | Array<{ type: string; text?: string }>;
};
```

The hook reads `session_id` to build the idempotency key and the recall lock path.

---

## Hook Response Format

### Stop / SubagentStop / PreCompact

These hooks return `{ decision: "continue" }` or exit with code 0. Claude Code ignores their stdout except for errors.

### PreToolUse

Returns JSON that Claude Code merges into context:

```json
{
  "assistantMessage": "<recalled-context>\n...\n</recalled-context>"
}
```

If no memories are found or no API key is set, the hook exits 0 with empty output — Claude Code proceeds normally.

## Enrichment Pipeline

### 1. `shouldCapture(messages)` — `enrichment/capture-filter.ts`

Filters out sessions that are not worth storing:
- Skips sessions with fewer than a minimum meaningful turn count
- Skips sessions where no assistant turn contains substantive content
- Returns `false` for greetings-only or error-only sessions

### 2. `extractDecisionChains(messages)` — `enrichment/chain-extractor.ts`

Identifies decision trees in the transcript:
- Finds assistant turns containing decisions, plans, architectures, or fixes
- Groups related turns into a chain summary
- Returns an array of `{ title, content, type }` objects

### 3. `redactSecrets(content)` — `enrichment/prompt-safety.ts`

Strips sensitive values before persistence:
- API keys matching common patterns (`sk-...`, `lano_...`, `ghp_...`, etc.)
- Bearer tokens and authorization headers
- `.env` style `KEY=VALUE` lines where the value looks like a credential
- Replaces matches with `[REDACTED]`

Also sanitizes recalled content against prompt injection patterns before injecting into Claude's context (`sanitizeForInjection()`).

### 4. `extractTags(content)` — `enrichment/tag-extractor.ts`

Auto-generates tags from content:
- Extracts technology names, framework names, and common domain keywords
- Returns up to 8 tags as lowercase strings

### 5. `detectMemoryType(content)` — `enrichment/type-detector.ts`

Classifies the memory into one of:
- `"project"` — architecture decisions, plans, designs
- `"context"` — ongoing work context, session state
- `"knowledge"` — general technical knowledge captured from the session
- `"feedback"` — user corrections, preferences, behavior guidance

## Idempotency Key

The Stop handler prevents duplicate memories using a SHA-256 key:

```
sha256( scope + "|" + agentType + "|" + type + "|" + normalizedContent[:500] )
```

- `scope`: value of `LANONASIS_ORG_ID` / `LANONASIS_PROJECT_ID`, or `"default"` if neither is set
- `agentType`: hook metadata agent type, usually `claude-code`
- `type`: detected memory type (`project`, `knowledge`, etc.)
- `normalizedContent[:500]`: whitespace-normalized first 500 characters of the memory body

This key is passed as `idempotency_key` in the `createMemory` request. MaaS deduplicates on this field.

---

## Recall Lock

The recall lock prevents multiple recall searches within a single session (e.g., when Claude Code restarts or the hook fires multiple times for the same session):

- **Lock path**: `~/.lanonasis/.recall-lock/<session_id>`
- **Written by**: `PreToolUse` handler at first call
- **Cleared by**: `Stop` / `SubagentStop` handler at session end
- **Stale locks**: Automatically cleared on next `Stop` for the same session

## StopOptions

The `handleStop()` function accepts:

```typescript
type StopOptions = {
  maxMemoriesPerSession: number;  // Cap on createMemory calls (default: 5)
  projectId?: string;             // Overrides LANONASIS_ORG_ID
  agentType?: string;             // Metadata tag (default: "claude-code")
  hook?: "stop" | "subagent_stop";
};
```

These are set in the hook scripts generated by `install-hooks.ts` from environment variables at install time.

## API Client — Key Behaviors

### `LanonasisClient` (`client.ts`)

- Auth: `X-API-Key: <LANONASIS_API_KEY>` header on every request
- Scope: `X-Project-Scope: <orgId>` header — **omitted entirely** when no org ID is configured (prevents 400 errors)
- Base URL: `https://api.lanonasis.com` (configurable via `LANONASIS_BASE_URL`)
- HTTP client: `undici` with `Agent({ connect: { family: 4 } })` — forces IPv4 to work around Node.js native fetch IPv6 issues

### `createMemory` Params

```typescript
{
  title: string;
  content: string;
  type: "project" | "context" | "knowledge" | "feedback";
  tags?: string[];
  metadata?: Record<string, unknown>;
  idempotency_key?: string;
}
```

### `searchMemories` Params

```typescript
{
  query: string;
  threshold?: number;   // 0–1, default from config (0.7)
  limit?: number;       // default 10
}
```

Returns array with `similarity` field (0–1) and either `type` or `memory_type` depending on MaaS version. The CLI handles both via:
```typescript
m.type ?? (m as Record<string, unknown>).memory_type ?? "unknown"
```

## Spool Queue — `SpoolQueue` (`spool.ts`)

On API failure, the Stop handler writes to the spool:

```typescript
spool.write({
  sessionId: string;
  memories: Array<{ title, content, type, tags, metadata }>;
  spooledAt: string;  // ISO timestamp
});
```

Each entry is appended as a newline-delimited JSON record. The `drain()` method iterates entries and retries `createMemory` for each.

**Spool directory structure:**
```
~/.lanonasis/maas-spool/claude-code/
└── <agent-type>.ndjson
```
